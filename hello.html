This single-file demo shows the files and terminal commands to reproduce the scenario described in Question 8: two developers (Alice and Bob) edit the same function in App.js on separate branches (feature1 and feature2), then both branches are merged into main causing a merge conflict that we resolve manually.

How to use

  1. Copy the terminal command section below and run it in an empty project folder (or adapt to your repo).
  2. Open the project in Visual Studio Code. When the conflict happens, use the Git -> Merge Editor or resolve by editing the file directly.
  3. After resolving, add, commit, and push the final result to your remote.

1) Initial repository & base App.js

Base file (on main initially):

// App.js (base on main)
function greet() {
  // This function will be modified by both Alice and Bob in different branches
  return 'Hello from main';
}

console.log(greet());

2) Alice's changes (branch: feature1)

Alice edits the function to add a name parameter and a friendly message:

// App.js (feature1 - Alice)
function greet(name) {
  return `Hello, ${name}! Welcome to our app. -- from Alice`;
}

console.log(greet('Alice'));

3) Bob's changes (branch: feature2)

Bob edits the same function differently (same function block modified):

// App.js (feature2 - Bob)
function greet(name) {
  // Bob wants a different default message and timestamp
  const who = name || 'Guest';
  return `Hi ${who}. Current year is ${new Date().getFullYear()}. -- from Bob`;
}

console.log(greet('Bob'));

4) What a merge conflict looks like (after merging both branches into main)

If you try to merge both commits into main without resolving, Git will mark App.js with conflict markers like this:

5) Resolved version (manual resolution)

Example of a manual resolution that merges both intentions: keep the parameter and defaulting behavior, include year and a friendly message, and attribute both authors in a comment.

// App.js (resolved)
function greet(name) {
  const who = name || 'Guest';
  // Combined message: friendly greeting + year + attribution
  return `Hello, ${who}! Current year is ${new Date().getFullYear()}. (Alice & Bob)`;
}

console.log(greet('Visitor'));
